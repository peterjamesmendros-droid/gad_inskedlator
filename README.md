## Tech Stack

- **Frontend/Backend:** Next.js 16.2.9 (App Router)
- **Database:** PostgreSQL via [Neon](https://neon.tech) (cloud-hosted, free tier)
- **Auth:** bcryptjs (passwords) + jose (admin JWT cookies)
- **Icons:** lucide-react v1.22.0
- **Font:** Poppins (Google Fonts)

- Node.js 18 or higher

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/peterjamesmendros-droid/gad_inskedlator.git
cd gad_inskedlator
```

### 2. Install dependencies

```bash
npm install
```

> **Important:** Always run `npm install` after cloning or pulling. If the `pg`, `bcryptjs`, or `jose` packages are missing, the database and authentication will not work.

### 3. Create `.env.local`

Create a file named **`.env.local`** in the root of the `gad_inskedlator` folder (same level as `package.json`). This file is never committed to Git.

```
POSTGRES_URL=
```

### 4. Run the schema (first time only)

If the database tables do not exist yet, run the contents of `gad_inskedlator_schema.sql` in the Neon SQL Editor to create all 8 tables.

### 5. Create the admin account (first time only)

Run this in your terminal from the project root to generate a bcrypt hash for your chosen password:

```bash
node -e "const b=require('bcryptjs'); console.log(b.hashSync('yourpassword', 10));"
```

Then run this in the **Neon SQL Editor**, replacing the hash with the one generated above:

```sql
INSERT INTO users (username, password, role)
VALUES ('admin', '$2b$10$YOUR_GENERATED_HASH_HERE', 'admin');
```

### 6. Start the development server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Common Issues

### ‚ùå "Database Write Error" or "ECONNREFUSED"

The app cannot connect to PostgreSQL. Check the following:

1. Does `.env.local` exist in the project root?
2. Is `POSTGRES_URL` set correctly with no extra spaces or line breaks?
3. Did you run `npm install`?
4. Did you restart the dev server after editing `.env.local`? Stop it with `Ctrl+C` and run `npm run dev` again ‚Äî Next.js only reads `.env.local` on startup.

### ‚ùå "relation does not exist"

The database tables have not been created. Run the schema SQL in the Neon SQL Editor (see Step 4 above).

### ‚ùå "Invalid credentials" on admin login

The password hash in the database was generated by a different bcrypt implementation. Generate a fresh hash using the project's own `bcryptjs` package (see Step 5 above) and update the row in Neon:

```sql
UPDATE users SET password = '$2b$10$YOUR_NEW_HASH' WHERE username = 'admin';
```

## Notes for Developers

- **`.env.local` is never committed to Git.** Each developer needs their own copy with the shared credentials.
- **Tailwind v4 rule:** In `globals.css`, `@import url(Google Fonts)` must come before `@import "tailwindcss"` or the build will fail.
- **User sessions** are stored in `localStorage` as `session_user` JSON `{ id, fullname, email }`.
- **Admin sessions** use an `admin_session_token` httpOnly cookie verified by `middleware.js`.
